Wei’s implementation for neighborhood extraction

New framework overview

The framwork is now divided into two parts:

  • Blocks extraction. Extract a sub-region in the physical layout. Generally, it is larger than the target window, and much smaller than the layout. Such a Divide-And-Conquer technique is memory-friendly, and easy to be paralleled
  • voxel extraction. For each block, we extract the physical voxels inside it.

To finish the voxel collection, we should run two parts separately.

Commands

Before all commands, it is recommended to update the database path --db_path in config.py to the correct one. Otherwise, should specify --db_path X at each run.

Block extraction

--stride: the stride (step size) of the window

--layer_id: the starting layer

--num_layer: number of layers in the sanwidch structure. For example, --layer_id 18 --num_layer 3 will extract blocks inside layer 18, 20, 22.

--bbox_size: the window size

--search_range: the search range to collapse voxels (like remove identical voxels)

Code organization (deprecated)

  • config.py: Some configurations, like database path (db_path), bounding box size (bbox_size) … can be specified by adding arguments
  • util.py: various functions for processing
  • main.py: main file

Framework (deprecated)

My framework is composed of several steps:

Steps input output related functions remarks
extract neighborhood locations for all rectangles poly_tensor a 2-D matrix bbox of the target layout area for each layer, each elements bbox[i][j] a list of polygonIDs that are intersected with or in the bbox. new_get_bbox GPU required!
PolygonID to NetID a 2-D matrix bbox For each element bbox[i][j], transfer from polygonIDs to NetIDs, remove the repeated NetIDs, and order the NetIDs. bbox_netid
Eq and Subset check a 2-D matrix bbox containing NetIDs Remove those elements whose NetIDs are 1) equal with some of the other elements or 2) are subset of some of the other elements iteratively_remove_eq_subset Re-implement with an efficient iterative manner. But may not be optimal (some subsets still exist in the result)
Netid to netname a list of element containing NetIDs transfer from NetIds to NetNames N/A
